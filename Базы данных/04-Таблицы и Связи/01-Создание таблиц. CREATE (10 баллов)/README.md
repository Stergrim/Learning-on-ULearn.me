# Создание таблиц. CREATE

Авиакомпания захотела вести блог, для этого нужно создать таблицу для хранения публикаций (articles).

| **Описание**            | **Название**   | **Тип**      | **Обязательность** | **Комментарий**                       |
|:------------------------|:---------------|:-------------|:-------------------|:--------------------------------------|
| ID                      | article_id     | Целое число  | +                  | Первичный ключ с автоинкрементом      |
| Заголовок               | title          | Строка       | +                  | Длина заголовка не более 255 символов |
| Текст                   | text           | Текст        | +                  |                                       |
| Автор                   | author         | Строка       | -                  | Длина ФИО автора не более 50 символов |
| Количество комментариев | comments_count | Целое число  | +                  | По умолчанию 0                        |
| Средняя оценка          | rating         | Число (10,2) | +                  | По умолчанию 0                        |
| Дата статьи             | article_date   | Дата и время | +                  |                                       |
| Дата создания           | created_date   | Дата и время | +                  | По умолчанию - текущее время          |
| Опубликовано            | published      | Да/Нет       | +                  | По умолчанию - Нет                    |

**Примечание**

Прежде чем начинать создавать таблицу крайне желательно иметь информацию по всем возможным ограничениям: типы полей, ограничения по длине, возможные значения, обязательность, значения по умолчанию, уникальность и т.п. Это сэкономит время и избавит от возможных ошибок в данных, от лишних изменений в БД.

**Задание**
1. [Освежи знания по документации](https://postgrespro.ru/docs/postgresql/13/ddl-basics).
2. [Посмотри синтаксис команды Create Table](https://postgrespro.ru/docs/postgresql/13/sql-createtable).
3. Создай таблицу articles.
4. Укажи правильные типы данных.
5. Задай ограничение обязательности полей.
6. Задай значения по умолчанию.
7. Вставь тестовые данные (да-да, это тоже часть задания).
8. Получи все статьи, отсортированные по ID. Обратите внимание на колонки со значениями по умолчанию: Число комментариев, Рейтинг, Дата создания, Публикация (и это тоже часть задания).

**Контрольные вопросы**
- Чем TIMESTAMP отличается от TIMESTAMPTZ
- Чем CHAR отличается от VARCHAR
- Какой тип подходит для хранения денег и почему.

**Все тесты пройдены, задача сдана:**
```pgsql
Create Table articles (
    article_id Serial Primary Key,
    title Varchar(255) Not Null,
    text Text Not Null,
    author Varchar(50),
    comments_count Integer Not Null Default 0,
    rating Numeric(10,2) Not Null Default 0,
    article_date Timestamp Not Null,
    created_date Timestamp Not Null Default Current_Timestamp,
    published Boolean Not Null Default False
);

Insert Into articles (title, text, author, article_date, published)
Values 
    ('Почему книги не горят', 'Текст статьи о книгах', 'Булгаков', '2021-01-01 00:00:00', True),
    ('В аэропорту Анадыря пропали три сестры', 'Текст статьи об исчезновении', 'Чехов', '2021-02-01 00:00:00', True),
    ('Зачем человеку паспорт?', 'Текст статьи о документах', 'Маяковский', '2021-03-01 00:00:00', False);

Select * From articles Order by article_id;
```
