# Минимизация данных

**Общие принципы**
- Выбирай только нужные строки, задавая условие WHERE.
- Выбирай нужное количество строк с использованием выражения LIMIT.
- Выбирай только нужные колонки из нужных таблиц. Не делай лишние JOINы. Не добавляй в запрос колонки большого размера, если они не нужны в приложении.
- Минимизируй количество запросов к БД.

**Задание**

Выполняй запросы в PGAdmin на демо-базе.
1. Посмотри план запроса `SELECT * FROM flights WHERE status='On Time' ORDER BY actual_arrival DESC;`
2. Создай индекс на колонку `actual_arrival`.
3. Посмотри план. Сравни с исходным. Почему такой тип доступа в плане?
4. Добавь ограничение на количество строк – выбери только первые 10 строк.
5. Посмотри итоговый план, сравни с предыдущими.
6. Удали индекс.
7. В ответ напиши основной тип доступа для каждого из 3х планов. В отдельных строках
